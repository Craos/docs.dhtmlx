What's New

Version 1.4.4

released on September 2, 2020

Fixes

  • Mechanism of appearing of Filters on a page is reworked
  • Rendering of lists in Filters is fixed

Version 1.4.3

released on May 27, 2020

Fixes

  • Fix the behavior of dragging and dropping in the data field which caused the position of selected fields not being changed
  • The footer content now influences on the alignment of the width of Grid columns
  • Mathematical calculations of fractional numbers in the footer of a grid are fixed
  • Fix the behavior of filters of fields when there are empty values in a data set
  • Fix the possibility to localize operations and dates
  • Fix the issue with rendering HTML content in cells of Grid

Version 1.4.2

released on March 5, 2020

Fixes

  • The issue with export data from Grid to Excel
  • The behavior of the setFields() method

Version 1.4.1

released on January 15, 2020

Fixes

  • Issue with attaching Pivot to Layout

Version 1.4

released on September 12, 2019

Breaking change

The pivot.export() method used for exporting Pivot data to Excel has been deprecated and will be removed in the next version. The API of the Export module will be used instead. Check the details in the Migration article.

New functionality

  • The ability to present grid data as a flat structure
  • Export to CSV format
  • Click-related and other Grid events are added into the Pivot API

Updates

  • The ability to change the appearance of fields selectors

Fixes

  • Issues with filters' popups in IE
  • Issues with footer values when customFormat is used

Version 1.3.3

released on January 10, 2019

Fixes

  • Issues with styles of cells in export data
  • Incorrect styles of filters' popups

Version 1.3.2

released on October 25, 2018

Fixes

  • Issues with null values in data
  • Issue with hiding filters

Version 1.3.1

released on August 8, 2018

Fixes

  • Issue with positioning of filters' popups

Version 1.3

released on July 10, 2018

Breaking change

There are minor changes in API: the rowsWidth property of the layout renamed to columnsWidth and the paint() method replaced the reload one. Check the Migration article for details.

New functionality

  • Ability to add a footer for total operations in columns
  • Ability to define templates for setting content of cells
  • Custom formatting for cell values
  • Autowidth for columns
  • A new grid with collapsible rows for rendering fields in a tree-like way

Version 1.2

released on November 2, 2017

Breaking change

Since version 1.2 the filtering API has been modified. Now the methods addFilter(), removeFilter() and getFilter() are deprecated in favor of a new extended API. Check the Migration article for details.

New functionality

  • UI for setting filters and extended filtering API
  • Ability to export Pivot to an Excel file
  • Ability to load data from external resources
  • Ability to use aliases in a data set to optimize the size of data
  • Conditional formatting of cells

Version 1.1

released on September 5, 2017

New functionality

  • Data sorting for columns and rows in both ascending and descending directions
  • Auto width for rows
  • Ability to define templates for headers
  • "Weeks" option for grouping by date
  • Ability to create custom options for grouping by date
  • Methods for manipulating filters: addFilter, getFilter, removeFilter

Version 1.0

released on August 10, 2017

Initial functionality

  • Ability to group and aggregate data
  • Ability to define grouping rules by drag-n-drop
  • Ability to define custom aggregation rules
